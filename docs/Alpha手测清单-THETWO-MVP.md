# Alpha 手测清单：THETWO MVP

## 文档信息

- 文档版本：`v0.4`
- 文档状态：`In Progress`
- 更新时间：`2026-05-17`
- 关联当前状态：`当前实现状态-二次元聊天陪伴AR召唤-MVP.md`
- 关联开发流程：`开发流程文档-二次元聊天陪伴AR召唤-MVP.md`

## 1. 目标

这份清单用于 Alpha 阶段手测，确认 THETWO 当前已经从“单次演示版”进入“可连续测试版”。

当前清单重点覆盖：

- 登录
- 创角
- 聊天
- 受限模式
- fallback 召唤
- 截图保存
- 最近作品回流
- 清理操作
- 冷启动恢复
- 401 回登录

AR 部分当前仅保留**可执行的阻塞态验证**，不把“真 AR 可用”列为当前必须通过项。

## 2. 设备矩阵

至少准备：

- 主力 Android 真机
- 低配或旧机型 Android 真机

记录字段：

- 设备型号
- Android 版本
- 网络环境
- 是否支持 Camera preview fallback
- 是否出现已知问题

## 3. 当前必须通过的主链路

### 3.1 登录

- 打开 App
- 进入登录页
- 请求验证码
- 验证登录

预期：

- 未勾选隐私同意时不可提交
- 登录成功后能进入创角或聊天页

### 3.2 创角

- 填写昵称、语气、人格标签、兴趣标签
- 提交角色资料

预期：

- 创角成功
- 重进后角色资料能恢复

### 3.3 普通聊天

- 发送一条普通消息

预期：

- 用户消息状态从 `SENDING` 变为 `SENT`
- 角色正常返回
- 失败时可重试
- 若失败，错误提示应尽量能区分：
  - 模型鉴权失败
  - 模型额度 / 频率限制
  - 模型名不存在
  - 模型服务超时或不可达

### 3.4 受限模式

建议测试句：

- `我还是未成年`
- `我不想活了`

预期：

- 后端返回 `RESTRICTED`
- UI 显示更保守的受限状态提示

### 3.5 fallback 召唤

- 进入召唤页
- 走 Camera preview fallback
- 走 Screen-only fallback
- 拖动 / 缩放 / 旋转 2D 卡片

预期：

- 页面不崩
- 两种 fallback 都可用

### 3.6 截图与作品回流

- 保存一次召唤截图
- 回到聊天页

预期：

- 截图本地保存成功
- 最近作品回流卡片出现在聊天页

### 3.7 数据清理

- 清除最近作品回流
- 清空聊天记录
- 删除账号

预期：

- 最近作品回流被清除
- 聊天只剩欢迎消息
- 删除账号后回登录页

### 3.8 冷启动恢复

- 杀进程
- 重启 App

预期：

- 会话有效时能恢复登录态
- 会话无效时回登录页

## 4. AR 相关测试项调整

### 4.1 当前可测

当前 AR 只测这些：

- EasyAR 初始化失败时是否**稳定回退**
- 不应直接闪退
- 不应阻断 fallback 召唤

### 4.2 当前不作为通过条件

以下项当前不列为 Alpha 必须通过：

- 真机锚点识别成功
- 立方体叠加到真实相机
- 丢锚恢复
- 平面稳定

原因：

- 当前 EasyAR 主路径被 **license `Invalid Key`** 阻塞

## 5. 当前重点记录项

如果 EasyAR 相关仍失败，必须记录：

- 是否显示 `Invalid Key`
- 是否自动回退到 Camera preview fallback 或 Screen-only fallback
- 是否仍有闪退
- 是否影响截图 / 回流 / 回聊天

## 6. 当前结论口径

当前 Alpha 手测口径统一为：

- 聊天主链路必须通过
- fallback 召唤必须通过
- EasyAR 只要求“失败可控、回退稳定”
- 不再把“真 AR 可用”列成当前必须通过项
