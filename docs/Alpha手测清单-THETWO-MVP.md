## 1. 目标

用于执行 P1 阶段的真机回归，确认 THETWO 已从“单次演示版”进入“可连续 Alpha 测试版”。

本清单只覆盖当前已实现主线：

- 登录
- 创角
- 聊天
- 受限模式
- fallback 召唤
- 截图保存
- 最近作品回流
- 数据清理
- 冷启动恢复
- 401 回登录

## 2. 设备矩阵

至少准备两台设备：

- 主力 Android 真机：支持相机预览，作为主回归设备
- 低配或旧机型 Android 真机：验证冷启动恢复、相机权限回退与页面稳定性

记录字段：

- 设备型号
- Android 版本
- 是否支持相机预览
- 网络环境
- 测试日期
- 测试人

## 3. 核心链路清单

### 3.1 首次登录

- 打开 App，进入 `Launch` 启动页后应跳转登录页
- 勾选隐私同意前，不能请求验证码或登录
- 输入邮箱后可请求验证码
- Debug 版能显示开发验证码提示
- 输入验证码后成功登录

预期结果：

- 登录成功后拿到会话
- `profileCompleted=false` 时进入创角页

### 3.2 创角

- 填写昵称、语气、人格标签、兴趣标签
- 提交后进入聊天页

预期结果：

- 创角成功
- 聊天首页标题显示当前角色名

### 3.3 普通聊天

- 发送一条普通消息
- 等待远端真实 LLM 回复

预期结果：

- 用户消息先显示 `SENDING`
- 成功后变成 `SENT`
- 角色回复正常返回
- Debug 日志出现 `chat_send_success`

### 3.4 受限模式

- 输入未成年人或危机内容测试句

建议测试句：

- `我还是未成年`
- `我不想活了`

预期结果：

- 后端返回 `RESTRICTED`
- 页面出现更保守的受限模式提示
- Debug 日志出现 `chat_restricted_mode_entered`

### 3.5 fallback 召唤

- 从聊天页进入召唤页
- 验证相机预览 fallback
- 验证纯屏 fallback
- 拖动、缩放、旋转角色卡片

预期结果：

- 页面不崩溃
- 两种 fallback 都可用
- Debug 日志出现 `summon_opened`

### 3.6 截图与作品回流

- 在召唤页保存截图
- 等待最近作品回流同步完成
- 返回聊天页

预期结果：

- 本地截图保存成功
- 最近作品回流卡片出现在聊天页
- Debug 日志出现：
  - `capture_saved_local`
  - `capture_sync_success`

### 3.7 清除最近作品

- 进入设置页
- 清除最近作品回流

预期结果：

- 聊天页和设置页都不再显示最近作品回流
- 系统相册截图文件仍保留
- Debug 日志出现 `recent_capture_cleared`

### 3.8 清空聊天

- 在设置页清空聊天记录

预期结果：

- 聊天页只保留欢迎消息
- 远端历史被清空
- Debug 日志出现 `chat_history_cleared`

### 3.9 删除账号

- 在设置页执行删除账号

预期结果：

- 返回登录页
- 本地会话、角色资料、最近作品回流全部清空
- 系统相册截图文件不删除
- Debug 日志出现 `account_deleted`

### 3.10 冷启动恢复

- 登录并完成创角后，完全杀进程
- 重新打开 App

预期结果：

- 进入 `Launch` 启动页
- 自动恢复登录态
- 若远端会话有效，直接进入聊天页或创角页
- 不要求重新登录
- Debug 日志出现：
  - `session_restore_started`
  - `session_restore_succeeded`

### 3.11 401 回登录

- 使用失效 `sessionToken` 或手工制造 401 场景

预期结果：

- 任一受保护接口返回 401 后，统一回到登录页
- 本地账号态被清理
- `arPrivacyAccepted` 保留
- Debug 日志出现 `session_unauthorized_cleared`

## 5. 当前通过标准

这一轮 Alpha 手测完成后，至少满足：

- App 重启后不会掉登录态
- 401 不会留下“本地还像登录着”的错觉
- 聊天、召唤 fallback、截图、回流、设置清理链路全部能在真机走通
- Debug 日志能覆盖登录、聊天、召唤、截图、回流和清理动作
