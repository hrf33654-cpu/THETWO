# 架构决策记录 007：百科词条创角只生成可编辑草稿

## 状态

`Accepted`

## 背景

当前 THETWO 的创角流程要求用户手动输入角色昵称、语气、人格标签和兴趣标签。这个流程可控，但首次使用成本偏高。用户希望通过百科词条选择来降低创角门槛。

百科数据源可以提供角色、作品或人物的公开摘要，但它们不是 THETWO 的内部角色设定库。直接复制百科内容会带来版权、内容安全、真实人物人格权、知名 IP 误导和外部接口稳定性风险。

## 决策

THETWO 可以支持“百科词条辅助创角”，但它只用于生成可编辑的 `CompanionProfile` 草稿，不直接创建最终角色。

固定流程：

- 用户输入关键词。
- 后端搜索外部词条候选。
- 用户选择一个候选词条。
- 后端获取短摘要或结构化信息。
- 后端调用 LLM 生成 THETWO 内部角色资料草稿。
- 客户端回填创角表单。
- 用户确认或修改后，才调用现有 `PUT /me/companion-profile` 保存。

技术边界：

- Android 客户端不得直连外部百科 API；原因是国内网络可用性、外部接口超时、API Key 泄露、限流、缓存和数据源替换都必须由后端统一控制。
- 外部 API Key 只允许放在后端环境变量。
- 不保存百科全文。
- 不展示大段百科正文。
- 不把百科角色包装成“官方角色”。
- 不改当前 `CompanionProfile` 主结构。
- 不影响手动创角流程。

第一版数据源优先级：

- `Bangumi` 作为首选二次元角色/作品候选源。
- `百度百科官方 API` 作为后续国内通用兜底源，前提是确认 API Key、额度、成本和使用条款。
- `萌娘百科` 暂不作为第一版主源，原因是授权、访问稳定性和社区内容风险更高。
- `Wikipedia / Wikidata` 只可作为服务端可选增强源，不得作为国内客户端依赖。

如后续实现，建议新增后端接口：

- `GET /profile-source/search?keyword=xxx`
- `POST /profile-source/generate-draft`

这两个接口只返回候选和草稿，不直接保存角色。

## 后果

收益：

- 降低首次创角输入成本。
- 保留用户确认和编辑权，避免错误词条直接污染角色资料。
- 后端统一隔离外部数据源，便于超时、限流、缓存、安全过滤和数据源替换。
- 不破坏现有创角保存协议。

代价：

- 需要维护外部数据源适配层。
- 搜索命中质量依赖外部平台。
- 需要额外处理版权、真实人物、成人内容、未成年人敏感内容和知名 IP 误导风险。
- 生成草稿仍需要 LLM，失败时必须回退到手动创角。

## 当前适用范围

- 本 ADR 只定义未来“百科词条辅助创角”的架构边界。
- 当前 P0/P1 主线仍优先做聊天侧摘要、基础记忆、安全状态和 Alpha 稳定性。
- 本决策不要求立即实现百科创角。
- 如果后续要把百科来源保存到 `companion_profiles`，需要单独评估是否增加 `source_type`、`source_id`、`source_url`、`source_title` 等可选字段。
