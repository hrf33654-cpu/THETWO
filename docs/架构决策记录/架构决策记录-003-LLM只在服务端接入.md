# 架构决策记录 003：LLM 只在服务端接入

## 状态

`Accepted`

## 背景

THETWO 后续需要接入真实 LLM、prompt、角色资料、会话摘要、安全策略、限流、日志和成本控制。如果 Android 客户端直接调用模型服务，会暴露 API Key，并导致安全策略、prompt 和模型切换难以统一管理。

当前工程已经有后端 `POST /chat/send`，Android 客户端通过后端发送聊天消息。

## 决策

LLM 只在服务端接入。Android 客户端不得直连 LLM 服务。

调用链固定为：

```text
Android -> THETWO 后端 -> LLM 服务
```

客户端只感知：

- 发送消息。
- 等待回复。
- 收到回复。
- 失败重试。
- 收到后端返回的 `NORMAL / RESTRICTED` 安全模式。

## 后果

收益：

- API Key 不进入客户端包。
- prompt、记忆、安全策略由服务端统一控制。
- 后续换模型或加审核不需要强制发新版 App。
- 可以在服务端做限流、日志、成本控制和失败兜底。

代价：

- 后端责任更重。
- 后端必须处理 LLM timeout、空回复、错误和安全拒答。
- 非流式与未来 streaming 的兼容需要在服务端设计。

## 当前适用范围

- 适用于聊天、记忆摘要、安全模式和未来所有模型能力。
- 任何客户端直连模型服务的方案都需要新 ADR 审批。
