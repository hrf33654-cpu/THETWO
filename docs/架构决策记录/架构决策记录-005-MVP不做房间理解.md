# 架构决策记录 005：MVP 不做房间理解

## 状态

`Accepted`

## 背景

房间扫描、空间建模、物体识别、空间记忆和持久 AR Anchor 都会显著增加开发复杂度、隐私风险、性能压力和失败路径。THETWO 当前 MVP 的重点是聊天陪伴和召唤回流，不是完整空间智能。

当前产品已经通过锚点图、相机叠加、截图保存和聊天回流形成可演示闭环。

## 决策

MVP 不做房间理解。

当前明确不做：

- 房间扫描。
- 空间建模。
- 物体识别。
- 空间语义理解。
- 空间记忆。
- 持久 AR Anchor。
- 基于真实房间结构的角色行为。

MVP 只做：

- 官方锚点图。
- CameraX 或纯屏 fallback。
- 截图保存。
- 最近作品回流。
- 聊天中引用最近作品上下文。

## 后果

收益：

- 降低隐私和权限解释成本。
- 降低 AR 技术风险。
- 聚焦聊天主闭环和 Alpha 可测性。
- 失败矩阵更可控。

代价：

- 角色不会真正理解用户房间。
- AR 表现阶段性停留在锚点或叠加体验。

## 当前适用范围

- 适用于 MVP 和 Alpha。
- 如果未来要引入空间理解，必须先新增隐私设计、数据模型、失败矩阵和 ADR。
