# 架构决策记录 004：优先交付 Fallback 召唤

## 状态

`Accepted`

## 背景

ARCore 图像锚定依赖设备、服务安装、相机权限、锚点图质量和 3D 资产稳定性。MVP 阶段如果先死磕真 AR，容易推迟可测试版本。

当前工程已经实现 CameraX fallback、纯屏 fallback、角色卡片拖动缩放旋转、截图保存和聊天回流。这条路径已经能验证“召唤 -> 截图 -> 回到聊天”的产品闭环。

## 决策

优先交付 fallback 召唤闭环，并将其作为 Alpha 可交付能力。真 AR 作为后续增强，不阻塞聊天主闭环和 Alpha 测试。

fallback 不是废弃方案，也不是临时调试页；它是 MVP 阶段正式支持的降级体验。

## 后果

收益：

- 大多数设备可以完成召唤体验。
- 截图回流和聊天上下文可以提前验证。
- ARCore 和 3D 资产风险不会阻断 Alpha。

代价：

- 早期召唤表现不是真实 AR 锚定。
- 后续接入 ARCore 时需要保证不破坏 fallback。

## 当前适用范围

- 适用于 MVP 到 Alpha 阶段召唤功能。
- 如果后续 AR 交互方案变化，应先保留 fallback，再通过新 ADR 调整主路径。
