# 架构决策记录 001：聊天是主闭环

## 状态

`Accepted`

## 背景

THETWO 同时包含聊天陪伴、角色召唤、截图回流和 AR 增强能力。AR 和视觉表现容易吸引注意力，但用户日常留存的核心来自持续可聊、可记忆、可失败恢复的角色关系。

当前工程已经具备聊天首页、远端聊天历史、发送状态、失败重试和最近作品回流。真实 LLM、记忆摘要和安全策略仍未完成。

## 决策

聊天陪伴是 MVP 和 Alpha 的主闭环。真实 LLM、回复速度、上下文、记忆、安全模式、失败重试和聊天留存优先级高于 ARCore、3D 表现和复杂召唤效果。

AR、截图和作品回流必须服务聊天主线，而不是替代聊天主线。

## 后果

收益：

- 开发优先级更清晰，避免过早陷入 AR 技术细节。
- Alpha 测试可以先验证角色陪伴是否成立。
- 真实 LLM 和安全策略能更早暴露风险。

代价：

- 早期 AR 表现可能不够完整。
- 需要接受 fallback 召唤作为阶段性交付能力。

## 当前适用范围

- 适用于 MVP 到 Alpha 阶段。
- 如果未来数据显示召唤/AR 是更强留存核心，可以通过新 ADR 修订。
